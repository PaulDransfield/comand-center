2. Hosting Options

Think of "hosting" as: where does the system live, and who can access it? There are three options, each with a different trade-off between simplicity, access, and cost.

🖥 Option A — Your Computer Only

Cost: Free · Setup: 20 min · Access: This computer only

Files live on your laptop or desktop. You open them in your browser. Nothing is on the internet. Your team can't access it unless they're sitting at your computer.

Pros
  • Completely free
  • No internet needed
  • Most private
  • Easiest setup
Cons
  • Only on your computer
  • Team can't use it
  • Lost if computer breaks
⭐ RECOMMENDED

☁️ Option B — Free Cloud (Vercel)

Cost: Free · Setup: 60 min · Access: Any device, anywhere

Files are hosted on Vercel (a free hosting service). You get a real web address like bjorken.vercel.app. You and your team can open it on any phone or computer.

Pros
  • Free forever
  • Works on all devices
  • Team can access
  • Automatic backups
Cons
  • Needs internet
  • 60min setup
  • Needs GitHub account

🏢 Option C — Your Own Server

Cost: 50–200 kr/month · Setup: 3–4 hours · Access: Anywhere

A rented computer in a data centre (like Railway or Render) runs 24/7. The Python automation runs constantly — invoices are processed the moment they arrive.

Pros
  • Always running
  • Full automation
  • Your own domain
Cons
  • Monthly cost
  • More complex setup
  • Requires more tech

🔒 Option D — Private Network (NAS)

Cost: Free (if you have a NAS) · Setup: 2 hours · Access: Home/office network

If you have a Synology or QNAP NAS (network storage device), you can run the system on it. Accessible from anywhere via VPN.

Pros
  • Very private
  • No ongoing cost
  • Full control
Cons
  • Needs existing NAS
  • Complex networking

3. What to Install

Before you can run the system, your computer needs a few programs. Here's exactly what to install and where to get it.

Required for HTML files only (basic use)

  • A modern web browser — Chrome, Firefox, Safari, or Edge
    You almost certainly have this already.
  • An Anthropic API key — for AI features
    Go to console.anthropic.com → Sign up → API Keys → Create key. Costs roughly 0–5 kr per day of normal use.

Required for full automation (Python tools)

  • Python 3.10 or newer
    Download from python.org/downloads. During installation, tick "Add Python to PATH". Verify by opening Terminal (Mac) or Command Prompt (Windows) and typing python3 --version
  • Python libraries — run this command after Python is installed
    Open Terminal and paste: 
    pip3 install python-docx python-pptx reportlab watchdog

Required for Vercel deployment (cloud)

  • A GitHub account (free)
    Sign up at github.com
  • A Vercel account (free)
    Sign up at vercel.com — use your GitHub account to sign in

5. Local Setup — Your Computer

Step 1: Create the folder structure

On Mac: Open Terminal (press ⌘+Space, type "Terminal").
On Windows: Open Command Prompt (press Win+R, type "cmd").

mkdir bjorken-bcc
cd bjorken-bcc
mkdir dashboard automation config inbox processed flagged logs db reports
mkdir reports/output

Step 2: Copy all your files

Move the files from wherever they were downloaded into the folders above. All .html files go in dashboard/, all .py files go in automation/, and all .json files go in config/.

Step 3: Install Python libraries

pip3 install python-docx python-pptx reportlab watchdog

Step 4: Verify everything works

# Test Python libraries
python3 -c "from docx import Document; from pptx import Presentation; from reportlab.lib.pagesizes import A4; print('All libraries OK')"

# Run the test suite
cd automation && python3 test_suite.py
# You should see: ✅ ALL TESTS PASSED

Step 5: Open the dashboard

Double-click any .html file in the dashboard/ folder. It opens in your browser. Start with chat_interface.html.

🔑

Add your API key

The first time you open any dashboard, look for the API key field (usually in the top-right corner or sidebar). Paste your Anthropic API key there. It's saved in your browser — you only need to do this once per browser.

6. Vercel Setup — Free Cloud Hosting

Vercel is a free hosting service. Once set up, your system is accessible at a web address from any device. The HTML files work perfectly on Vercel. The Python automation needs to run on your computer or a server.

  1. 1

    Create a GitHub repository

    Go to github.com/new. Name it bjorken-bcc. Set it to Private. Click "Create repository".

  2. 2

    Upload your files to GitHub

    On the repository page, click "uploading an existing file". Drag your dashboard/ folder contents in. Click "Commit changes".

  3. 3

    Connect Vercel to GitHub

    Go to vercel.com/new. Click "Import Git Repository". Select your bjorken-bcc repository. Click "Deploy".

  4. 4

    Get your URL

    Vercel gives you a URL like bjorken-bcc.vercel.app. Share this with your team. Bookmark it on your phone.

  5. 5

    Add environment variables (API key)

    In Vercel → your project → Settings → Environment Variables. Add ANTHROPIC_API_KEY with your key value. This keeps your key out of the files.

  6. 6

    Test the deployment

    Open your Vercel URL. The dashboards should load. Enter your API key in the interface. Try asking a question in the chat.

⚠️

Python automation on Vercel

The HTML dashboards work on Vercel. The Python automation (workflow_engine.py, report_scheduler.py) needs to run somewhere with Python — your laptop, or a paid server. For now, run them manually on your computer when needed.

7. Configuration Files

workflow_config.json — Invoice processing rules

This is the main settings file. Open it in any text editor (Notepad on Windows, TextEdit on Mac).

The three things to change first:

  "name":    "Restaurang Björken AB",  ← your company name
  "org_nr":  "556XXX-XXXX"           ← your org number
},

"thresholds": {
  "food_pct_warn":   31.0,   ← alert when food costs exceed 31%
  "staff_pct_warn":  40.0,   ← alert when staff costs exceed 40%
  "profit_margin_warn": 12.0  ← alert when margin falls below 12%
},

"anthropic_api_key": "sk-ant-..."  ← paste your API key here

report_config.json — Report schedules and recipients

"schedules": {
  "daily": {
    "enabled":    true,
    "time":       "07:00",           ← what time to send
    "recipients": ["[email protected]"]  ← who gets the report
  }
},
"email": {
  "enabled":   true,
  "username":  "[email protected]",    ← your Gmail address
  "password":  "xxxx xxxx xxxx xxxx" ← Gmail App Password (not your main password)
}

9. Maintenance Guide

Daily tasks (5 minutes)

  • Check the daily summary email (arrives at 07:00)
    Look for any RED alerts — these need action today
  • Check the flagged/ folder
    Any files here need manual review — unknown suppliers or unusual amounts

Weekly tasks (15 minutes)

  • Review the weekly report (sent Monday 08:00)
    Particularly check cost percentages vs targets
  • Check reconciliation — unpaid invoices tab
    Any invoices showing as overdue need to be paid or queried
  • Import bank CSV (if not using automatic bank sync)
    Log in to Handelsbanken → download statement → drag into Reconcile tab

How to back up your data

All your important data lives in two places:

WhatWhereHow to back up
Invoice database db/workflow.db Copy this file to Google Drive or Dropbox weekly
Processed invoices processed/ folder Copy folder to backup drive monthly
Generated reports reports/output/ Already PDFs — copy to Google Drive monthly
Configuration config/ folder Copy after any changes to settings

Automated backup (Mac/Linux)

# Add this to your crontab (run: crontab -e) for daily backup at midnight
0 0 * * * cp ~/bjorken-bcc/db/workflow.db ~/bjorken-bcc/db/backup_$(date +%Y%m%d).db

Restore from backup

# Stop the workflow engine first, then restore
cp ~/bjorken-bcc/db/backup_20260301.db ~/bjorken-bcc/db/workflow.db
# Restart the workflow engine

How to update when you want new features

When new versions of the files are created (either by me building more features, or by you editing them), here's how to update safely:

  1. 1

    Back up first

    Copy the db/ and config/ folders to your desktop before changing anything.

  2. 2

    Replace only the files that changed

    Don't replace workflow_config.json or report_config.json — those have your personal settings. Only replace .py and .html files.

  3. 3

    Run the tests

    python3 test_suite.py — if all pass, the update is good. If tests fail, restore from backup and report the issue.

11. User Manual — Uploading Documents

The system can read invoices, P&L statements, bank statements, contracts, and any business document. Here's how to add them.

Method 1: Drag and drop into the browser (instant, for chat Q&A)

  1. 1

    Open rag_system.html or chat_interface.html

    Any of the HTML dashboards with a document panel on the left side.

  2. 2

    Click "Upload documents" or drag files in

    Supported formats: PDF, Excel (.xlsx), CSV, text files (.txt). You can upload multiple files at once.

  3. 3

    Wait for indexing

    A chunk count appears next to each document (e.g. "12 chunks indexed"). This takes 1–5 seconds. Once you see the green "Ready" badge, the document is searchable.

  4. 4

    Ask a question

    Type any question about the document. The RAG Inspector panel (right side) shows exactly which parts of which documents were used to answer.

Method 2: Drop into inbox/ folder (for automatic processing)

When the workflow engine is running, any file dropped into the inbox/ folder is automatically:

  • Read and data extracted (vendor, amount, date, line items)
  • Categorised (food / staff / rent / other)
  • Added to the financial tracker
  • Checked for anomalies
  • Moved to processed/ (or flagged/ if there's an alert)
📝

Best format for invoices

The system reads text best. If you have a PDF that was scanned (a photo of paper), the text extraction may be imperfect. For best results, use PDFs exported from accounting software, or copy-paste the text into a .txt file.

Types of questions and how to ask them

Question typeExampleTip

Analysis "Are our food costs higher than industry average?" The system compares against built-in industry benchmarks
Planning "What should we budget for Q2 food costs?" Based on actual March data and trends

Good questions to ask regularly

  • "What were our top 5 expenses last month?"
  • "Are there any invoices due in the next 7 days?"
  • "Which of our cost categories is furthest over budget?"
  • "Summarise all documents uploaded this week"
  • "What is our profit margin trend over the last 3 months?"
  • "Create a monthly summary report for the board"

Understanding the citations

After each AI answer, you'll see blue chips showing which documents were used: [1] PL_Mars_2026.txt. Click any chip to highlight the relevant document in the library panel. This tells you exactly where the answer came from — you can verify the original document any time.

💡

The AI only knows what you've uploaded

If you ask about something and the AI says "I don't have information about that in your documents," it means the relevant document hasn't been uploaded yet. Upload it and ask again.

14. User Manual — Reading the Tracker

The four KPI cards

At the top of most dashboards are four key numbers. Here's how to read them:

CardWhat it showsGood rangeWarning sign
Revenue Total sales this period ↑ trending up Down more than 10% vs last month
Food & Bev % Supplier costs as % of revenue Below 31% Orange: above 31% · Red: above 35%
Staff % All payroll as % of revenue Below 40% Orange: above 40% · Red: above 50%
Net Profit % What's left after all costs Above 12% Orange: below 12% · Red: below 7%

Colour coding everywhere

  • 🟢 Green — Within target, all good
  • 🟡 Amber — Approaching limit, worth watching
  • 🔴 Red — Over limit, action needed today

What to do when you see a red alert

  1. 1

    Read the alert detail

    Click or hover the red badge. It tells you exactly what the issue is: "Food costs at 33.2% — above 31% target".

  2. 2

    Ask the chat why

    Type: "Why are food costs above target this month?" The AI looks at your invoices and explains.

  3. 3

    Act on the recommendation

    The chat suggests specific actions (e.g. "Menigo invoice in March was 8% higher than usual — consider renegotiating contract").

15. When Things Don't Work

Quick diagnostic checklist

  • Is the browser showing any error message? Screenshot it.
    Press F12 → Console tab to see technical error details
  • Is the workflow engine running in Terminal?
    You should see "[timestamp] Watching inbox/ for new invoices…" — if not, restart it
  • Has the API key expired or run out of credit?
    Check console.anthropic.com → Usage dashboard
  • Was a configuration file accidentally edited?
    Restore from your config/ backup

Emergency: complete system reset

If nothing works, this restores the system to a clean state without losing your invoice data:

# 1. Back up your data first
cp -r db/ db_backup_$(date +%Y%m%d)
cp -r config/ config_backup_$(date +%Y%m%d)

# 2. Reinstall Python libraries
pip3 install --upgrade python-docx python-pptx reportlab watchdog

# 3. Run the test suite to identify what's broken
python3 test_suite.py
# Look for ✗ marks — these tell you exactly which part failed

16. Security — API Key Storage

Your Anthropic API key is like a credit card. Anyone who has it can use it and it charges to your account. Here's how to store it safely.

Do this

  • Store it in the browser (saved by the dashboard interface) — safe
  • Store it in workflow_config.json — acceptable for local use only
  • Use environment variables on Vercel/servers — best practice
  • Set a monthly spending limit at console.anthropic.com → Billing → Usage limits

Never do this

  • Send the API key in an email
  • Put it in a file you upload to GitHub (even a private repository)
  • Share it with anyone outside your business
  • Leave it in a public Google Doc
🚨

If your API key is compromised

Go to console.anthropic.com/settings/keys immediately. Delete the compromised key. Create a new one. Update it in all your config files.

17. Data Privacy

Your financial data is sensitive. Here's exactly what goes where:

DataWhere it goesPrivacy level
Invoice content (amounts, vendors) Stays on your computer / your database 🟢 Private — never leaves your system
Questions you ask the AI Sent to Anthropic Claude API (encrypted) 🟡 Anthropic's data policy applies

Fortnox connection data Stays on your computer (OAuth tokens) 🟢 Private — never leaves your system

GDPR consideration

If your documents contain personal data about employees (names, salaries, personal ID numbers), be cautious about what you send to the Claude API. The AI doesn't need to see employee names to answer financial questions — you can redact names from documents before uploading.
